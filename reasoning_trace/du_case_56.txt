Cross-Component Correlations and Root Cause Hypothesis:
Timelines: CU starts F1AP at ~same time DU asserts and exits; UE tries connect immediately but fails repeatedly. DU crash prevents F1AP completion and rfsim server startup, causing UE connect failures; CU GTPU bind failure secondary (address issue). Misconfigured_param "gNBs[0].servingCellConfigCommon[0].dl_UL_TransmissionPeriodicity=0" causes issue: In OAI, dl_UL_TransmissionPeriodicity=0 maps to 0.5ms periodicity (tdd_ms_period_pattern[0]=0.5), at SCS 30kHz (mu=1), slots/frame=10, periods/frame=10ms/0.5ms=20, slots/period=10/20=1. But config specifies nrofDownlinkSlots=7, nrofUplinkSlots=2, nrofMixedSlots=1 (total 10), expecting ms5 periodicity (slots/period=10). Assertion in set_tdd_config_nr checks nb_slots_per_period == sum(slots)+1, but 1 != 10+1, fails. Hypothesis: dl_UL_TransmissionPeriodicity=0 invalid for this slot config; should be 6 (ms5, tdd_ms_period_pattern[6]=5.0). Impacts: DU early exit on config validation, F1AP incomplete, no rfsim server, UE connect fails. External knowledge: 3GPP TS 38.213 Table 11.1-1 defines periodicity enums; OAI maps 0 to 0.5ms, but slot counts must match computed slots/period.
